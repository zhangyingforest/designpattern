设计模式之迪米特法则（Demeter Principle）  最小知识原则

迪米特法则在于降低类之间的耦合，每个类尽量减少对其他类的依赖，尽量减少对外暴露的方法，使得功能模块独立，低耦合

理解：
    a. 只与直接的朋友交流（成员变量、方法的输入输出参数中的类）；
    b. 减少对朋友的理解（减少一个类对外暴露的方法）。
注意：
    a. 虽然可以避免和非直接的类通信，但是要通信，必然会通过一个”中介“来发生联系，
       过分的使用迪米特原则，会产生大量的中介和传递类，导致系统复杂度变高。


总结和注意事项

1、迪米特法则的核心是降低类之间的耦合
2、但是注意：由于每个类都减少了不必要的依赖，因此迪米特法则只是要求降低类间(对象间)耦合关系， 并不是要求完全没有依赖关系
